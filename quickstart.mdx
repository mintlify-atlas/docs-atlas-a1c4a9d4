Get your voice-powered research assistant up and running quickly. This guide walks you through installation, setup, and your first question.

## Installation

Klaus is available via Homebrew for macOS and pipx for Windows. Both methods handle all dependencies automatically.

<Tabs>
  <Tab title="macOS (Homebrew)">
    ```bash
    brew tap bgigurtsis/klaus
    brew install klaus
    ```
  </Tab>
  
  <Tab title="Windows (pipx)">
    First install pipx if you don't have it:
    ```bash
    pip install pipx && pipx ensurepath
    ```
    
    Restart your terminal, then install Klaus:
    ```bash
    pipx install klaus-assistant
    ```
  </Tab>
  
  <Tab title="From Source">
    <Note>
      **macOS:** Requires Python 3.11-3.13 and PortAudio
      
      **Windows:** Requires Python 3.11-3.13 and [Visual C++ Build Tools](https://visualstudio.microsoft.com/visual-cpp-build-tools/)
    </Note>
    
    Install prerequisites on macOS:
    ```bash
    brew install python@3.13 portaudio
    ```
    
    Clone and install Klaus:
    ```bash
    git clone https://github.com/bgigurtsis/Klaus.git
    cd Klaus
    pip install -e .
    ```
  </Tab>
</Tabs>

<Warning>
**macOS 26 + Python 3.14:** Global hotkeys can crash on this combination. Klaus automatically disables global hotkeys and keeps in-app hotkeys active. Use Python 3.13 for stable global hotkeys.
</Warning>

## First Launch

Launch Klaus from your terminal:

```bash
klaus
```

On first launch, you'll see a 7-step setup wizard that configures everything you need.

<Steps>
  <Step title="Welcome">
    Read the introduction to Klaus and click Next to begin setup.
  </Step>
  
  <Step title="API Keys">
    Enter your API keys from three providers:
    
    - **Anthropic** (vision + reasoning): [console.anthropic.com](https://console.anthropic.com/settings/keys)
    - **OpenAI** (text-to-speech): [platform.openai.com](https://platform.openai.com/api-keys)
    - **Tavily** (web search, free tier 1,000/mo): [app.tavily.com](https://app.tavily.com/home)
    
    <Note>
    **macOS:** Keys are stored securely in Apple Keychain. Any existing plaintext keys in `~/.klaus/config.toml` are automatically migrated.
    
    **Windows:** Keys are stored in `~/.klaus/config.toml`.
    </Note>
  </Step>
  
  <Step title="Camera Selection">
    Choose your camera device. Klaus needs a camera to see what you're reading.
    
    A USB document camera (visualiser) or phone on a gooseneck mount works best. Klaus auto-detects portrait orientation and rotates the image.
  </Step>
  
  <Step title="Microphone Test">
    Select your microphone and test the audio level. Speak into the mic and verify the level indicator responds.
  </Step>
  
  <Step title="Voice Model Download">
    Klaus downloads [Moonshine](https://github.com/usefulsensors/moonshine) Medium (~245MB), a local speech-to-text model. This happens on first use and runs entirely on your device with no API cost.
  </Step>
  
  <Step title="About You (Optional)">
    Add your background, research focus, or reading interests. Klaus uses this context to tailor responses.
    
    You can also configure your Obsidian vault path here for hands-free note-taking.
  </Step>
  
  <Step title="Setup Complete">
    Click "Start Klaus" to launch the main application.
  </Step>
</Steps>

<Note>
**macOS input monitoring:** macOS may prompt you to grant your terminal Accessibility (input monitoring) permission. This is needed for global hotkeys to work when Klaus is not focused. You can deny the prompt and use the in-app UI buttons instead.
</Note>

## Ask Your First Question

Once Klaus is running:

1. **Place a document** under your camera so Klaus can see the page
2. **Ask a question** out loud or press the PTT key (default: `F2`)
3. **Wait 2-4 seconds** for Klaus to read the page and respond

Klaus captures the page image, sends it to Claude's vision API along with your transcript, reasons about the question, and responds aloud through text-to-speech.

## Input Modes

Klaus supports two ways to interact:

<CodeGroup>

```text Voice-Activated (Default)
Simply start speaking. Klaus detects your voice automatically via WebRTC VAD.
After a brief silence (~1.5s), it finalizes your question and starts processing.

Toggle key: § (macOS) or F3 (Windows)
```

```text Push-to-Talk
Hold the PTT key (default F2) to record.
Release to send your question.

Configure hotkey in ~/.klaus/config.toml
```

</CodeGroup>

Toggle between modes anytime:
- Press the toggle key (`§` on macOS, `F3` on Windows)
- Click the mode button in the UI

<Note>
On macOS, F-keys trigger system actions by default (F3 = Mission Control). Press `Fn + F3` to use the toggle, enable "Use standard function keys" in System Settings, or configure a different key in `~/.klaus/config.toml`.
</Note>

## Updating Klaus

Keep Klaus up to date with the latest features and fixes:

<CodeGroup>

```bash macOS
brew upgrade klaus
```

```bash Windows
pipx upgrade klaus-assistant
```

```bash From Source
cd Klaus
git pull
pip install -e .
```

</CodeGroup>
